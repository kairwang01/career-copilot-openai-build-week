06 — Governed AI execution
Reliability behind the interface

Every request follows a server-owned path.

Authenticate

Confirm identity and role before execution.

02 / LIMIT

Validate

Bound the payload and reject unsafe shapes.

03 / ONCE

Claim once

Use an idempotent request identifier.

04 / POLICY

Route

Select an allowed model through server policy.

05 / SCHEMA

Verify output

Check structured results before the UI uses them.

06 / RECOVER

Record or refund

Persist usage and recover failed charges safely.
